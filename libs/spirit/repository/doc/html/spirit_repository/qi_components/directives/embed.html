Qi Embed Parser Directive

Description

The embed[] parser-directive provides a mechanism for parsing against the embedded range instead of normal input.

A Range is a general concept, such as STL container, std::string, and boost::iterator_range are all ranges. For Spirit.Qi, it's natively boost::iterator_range, as what you get from qi::raw[].

Spirit.Qi requires Forward Range. There's no restriction imposed on the range type, even whiches range-iterator not compatible with input iterator can be used, while it actually depends on the enclosed parser.

Header 
// forwards to <boost/spirit/repository/home/qi/directive/embed.hpp>
#include <boost/spirit/repository/include/qi_embed.hpp>

Also, see Include Structure.

Example

The following example shows a simple use case of the embed[] directive that exploits Nabialek-Trick and demonstrates the basic error-handling ability. (for the full example code, see embed.cpp)

Some namespace aliases & using declaration: 

namespace qi = boost::spirit::qi;
namespace phx = boost::phoenix;
namespace repo = boost::spirit::repository;
using namespace boost::spirit::standard;

And some typedefs: 

typedef boost::variant<int, double, std::string> V;
typedef std::string::const_iterator iter;
typedef qi::rule<iter, V()> rule_t;
typedef boost::iterator_range<iter> range;

Here we declare the rules and symbol table: 

qi::rule<iter, V(), qi::locals<range, rule_t*>, space_type> demo;
qi::rule<iter, range()> kwd_patt;
qi::rule<iter, rule_t*()> full_kwd;
qi::symbols<char, rule_t*> kwd; 
rule_t ri, rd, rs;

Let's define our rules now: 

demo %= 
    qi::omit
    [
        kwd_patt                                  [qi::_a = qi::_1]
     >> repo::qi::embed(qi::_a)[full_kwd]         [qi::_b = qi::_1]
    ]
  > qi::lazy(*qi::_b)
    ;

// matches whole string without trailing chars
full_kwd = kwd > qi::eoi;
 
kwd_patt = // the keyword pattern
    qi::raw[!qi::digit >> +(qi::alnum | '_')][qi::_val = qi::_1]
    ;

In demo, we try to match the keyword pattern kwd_patt, store the matched range in the local variable qi::_a, and feed full_kwd with it through the embed[] directive. After full_kwd matched, a corresponding rule is stored into the local variable qi::_b (by pointer), which we expect to match after seeing the keyword.

full_kwd expects a full-match of kwd.

kwd_patt matches string like C++ identifier, exposing the matched range.

Here's the rest of our rules: 

ri = qi::int_;
rd = qi::double_;
rs = qi::as_string['"' >> *~qi::char_('"') >> '"'];
    
kwd.add("int"      , &ri)
       ("double"   , &rd)
       ("string"   , &rs)
    ;

For error-handling: 

ri.name("int");
rd.name("double");
rs.name("string");

qi::on_error<qi::fail>
(
    demo
  , std::cout
        << phx::val("[demo] Error! Expecting ")
        << qi::_4                                       // what failed?
        << phx::val(" here: \"")
        << phx::construct<std::string>(qi::_3, qi::_2)  // iterators to error-pos, end
        << phx::val("\"")
        << std::endl
);

qi::on_error<qi::fail>
(
    full_kwd
  , std::cout
        << phx::val("[full_kwd] Error! Expecting ")
        << qi::_4                                       // what failed?
        << phx::val(" here: \"")
        << phx::construct<std::string>(qi::_3, qi::_2)  // iterators to error-pos, end
        << phx::val("\"")
        << std::endl
);

Since we use expectation-points in both the main rule demo and the embedded full_kwd, we'd better define the error-handlers for them respectively.
